## 1. v1.0 定义

v1.0 不是云端 SaaS，也不是多人协作系统。

v1.0 的目标是：

1. 用户可以在本机打开 `LinoI.app`。
2. 本地后端稳定运行在 `http://127.0.0.1:7773`。
3. 用户可以创建一本书，导入前三章。
4. 系统能调用已配置的大模型分析前三章，生成初始基础文件：
   - World Bible
   - Character Cards
   - Memory / Chapter Facts
   - Knowledge Matrix
5. 用户可以从第 4 章开始，稳定跑完五步章节工作流：
   - 输入 Prompt
   - 审核结构化 Prompt
   - 审核正文 / 请求修改
   - 批准正文
   - 确认基础文档更新
6. 本地数据不会轻易丢失，有备份和恢复路径。
7. 大模型失败、网络失败、JSON 解析失败、后端失败时，用户能看到明确错误并重试。

## 2. v1.0 暂不做

以下内容不作为 v1.0 阻塞项：

- 云部署。
- 域名、Nginx、HTTPS、远程 Docker。
- 多用户、登录系统、账号体系。
- App Store 上架。
- 自动更新。
- 多设备同步。
- pgvector 检索生产化。
- 完整可观测性平台。
- 出版排版、连载发布、多人协作。

## 3. 当前基线

当前已具备：

- `NovelOSMac` SwiftUI/macOS App，打包名已改为 `LinoI.app`。
- 本地 FastAPI 后端。
- Novel CRUD。
- 前三章导入与分析接口。
- LLM Provider 配置。
- 本地 LaunchAgent：`top.linotsai.novelos.local7773`。
- 五步 Chapter Studio 工作流。
- Base Documents 与 Knowledge Matrix 基础编辑能力。
- 后端测试和 Swift 测试。

当前已验证命令：

```bash
cd NovelOSBackend
.venv/bin/python -m pytest

cd ../NovelOSMac

Scripts/package_app.sh
```

## 4. v1.0 上线步骤

### Step 1：冻结 v1.0 范围

目标：避免边做边膨胀。

必须完成：

- 明确 v1.0 只做本地 App。
- 保留五步主工作流，不增加聊天 UI。
- 保留“基础文件可手动编辑”。
- 新需求只进入 v1.1，除非它阻塞 v1.0 可用性。

验收：

- 本文件存在并作为后续任务基准。
- 后续 issue / TODO / 修改说明引用本文件的步骤编号。

---

### Step 2：稳定本地运行时

目标：用户不需要懂后端，也能稳定打开 App 使用。

必须完成：

- 本地后端默认地址固定为 `http://127.0.0.1:7773`。
- LaunchAgent 可启动、停止、重启、查看状态。
- App 启动时能检测后端是否可用。
- 后端不可用时，App 给出明确提示，而不是空白或假装正常。
- `.env`、owner token、LLM Provider 配置不进入 Git。

验收命令：

```bash
curl -fsS http://127.0.0.1:7773/healthz
launchctl print gui/$(id -u)/top.linotsai.novelos.local7773
cd NovelOSBackend
scripts/local_service.sh status
scripts/local_service.sh health
```

---

### Step 3：前三章导入分析生产化

目标：导入前三章后，必须真的生成可用 Canon 起点。

必须完成：

- `POST /api/novels/{novel_id}/bootstrap/import-first-three-chapters` 持久化三章原文。
- `POST /api/novels/{novel_id}/bootstrap/analyze` 调用 LLM。
- LLM 返回结果必须落库到：
  - World Bible
  - Character Cards
  - Memory Facts
  - Knowledge Matrix
- 分析结果支持幂等重跑，不重复堆脏数据。
- 模型 JSON 不完整时有 fallback 或清晰错误。
- App 导入后自动刷新四类基础文件。

验收：

```bash
curl -fsS http://127.0.0.1:7773/api/novels/<novel_id>/world-bible
curl -fsS http://127.0.0.1:7773/api/novels/<novel_id>/characters
curl -fsS http://127.0.0.1:7773/api/novels/<novel_id>/memory
curl -fsS http://127.0.0.1:7773/api/novels/<novel_id>/knowledge-matrix
```

四个接口都不能是空数组，除非用户明确导入空内容。

---

### Step 4：LLM 调用稳定性

目标：模型不稳定时，系统还能给用户一个可恢复的状态。

必须完成：

- 所有 LLM 调用记录：
  - provider id
  - model
  - input_json
  - output_json
  - token_usage
  - error_message
- 结构化输出统一校验。
- JSON fenced block、额外解释文本、字段缺失要能处理或报清楚。
- 超时、401、429、5xx 要显示明确原因。
- 用户可以重试失败的导入分析或章节生成。

验收：

- 关闭网络或填错 API key 后，App 显示明确错误。
- 恢复 API key 后，同一任务可以重试成功。
- 后端测试覆盖至少一个模型失败场景和一个脏 JSON 场景。

---

### Step 5：第 4 章到第 6 章真实流程压测

目标：验证系统不是只能跑一次 demo。

必须完成：

- 用同一本真实小说连续生成第 4、5、6 章。
- 每章都跑完整五步。
- 每章批准后生成 Canon Patch。
- Canon Patch 确认后，基础文件有合理更新。
- Knowledge Matrix 不出现明显自相矛盾。
- 人物卡不会被重复创建成多个同名人物。

验收：

- 至少完成一轮真实小说三章连续生成记录。
- 手动记录每章失败点、修改点、模型问题。
- 修掉所有 P0/P1 级阻塞。

---

### Step 6：基础文件编辑与版本安全

目标：用户手动修 Canon 不会变成不可恢复的破坏性操作。

必须完成：

- World Bible、Character Cards、Memory、Knowledge Matrix 的增删改稳定。
- 角色卡不做物理删除，至少保持当前“无 DELETE”策略。
- 基础文件保存失败时不吞错误。
- Canon Patch 确认动作必须写 `canon_edit_history`。
- 关键编辑支持追踪来源：manual / import_agent / canon_merge_agent。

验收：

- 手动编辑四类基础文件后，刷新 App 数据仍然存在。
- 后端数据库能查到相关 edit history。

---

### Step 7：本地数据备份与恢复

目标：v1.0 可以放心写长篇，不怕一天数据全没。

必须完成：

- 明确本地数据库位置。
- 提供一键备份脚本或命令。
- 提供恢复说明。
- 备份包含：
  - 数据库
  - 导入原文文件
  - `.env` 示例说明，但不自动复制密钥到公开位置
- App 或 README 明确提示备份路径。

验收：

```bash
# 示例，最终以实际脚本为准
cd NovelOSBackend
scripts/backup_local.sh
LINOI_DRY_RUN_RESTORE=1 scripts/restore_local.sh <backup_path>
LINOI_CONFIRM_RESTORE=1 scripts/restore_local.sh <backup_path>
```

---

### Step 8：App 体验收尾

目标：让用户知道系统在做什么，而不是“点了没反应”。

必须完成：

- 导入、分析、生成、审核、确认都有 loading 状态。
- 长耗时操作有明确状态文案。
- 空状态不误导用户。
- 失败状态有下一步按钮或可执行建议。
- App 名、图标、窗口标题统一为 `LinoI`。
- 打包产物固定为 `NovelOSMac/dist/LinoI.app`。

验收：

- 新建书到第 4 章准备完成，全程没有无反馈等待。
- 打包后 `Info.plist` 中显示：
  - `CFBundleDisplayName = LinoI`
  - `CFBundleName = LinoI`
  - `CFBundleExecutable = LinoI`

---

### Step 9：测试闸门

目标：每次准备 v1.0 包之前，必须通过固定测试。

必须通过：

```bash
cd NovelOSBackend
.venv/bin/python -m pytest

cd ../NovelOSMac
swift test
Scripts/package_app.sh
codesign --verify --deep --strict dist/LinoI.app
```

还必须手动验证：

- 创建新书。
- 导入前三章。
- 生成基础文件。
- 创建第 4 章。
- 跑完整五步工作流。
- 退出 App 后重开，数据仍在。
- 重启本地后端后，App 仍能连接。

---

### Step 10：v1.0 发布包冻结

目标：生成一个可回滚、可复现的本地版本。

必须完成：

- Git 工作区只包含 v1.0 相关改动。
- 写一份简短 release note。
- 打 tag：`v1.0.0-local`。
- 保留打包产物路径：`NovelOSMac/dist/LinoI.app`。
- 记录测试结果。
- 记录当前 LLM Provider 名称和模型名，但不记录 API key。

验收：

```bash
git status --short
git tag v1.0.0-local
```
